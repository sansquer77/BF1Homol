# Migration Notes: SQLite -> PostgreSQL 18

## Objetivo
Padronizar a aplicacao para uso exclusivo de PostgreSQL 18, removendo caminhos, sintaxe e fallback de SQLite, sem alterar regras de negocio.

## Escopo Executado
- Backend de banco definido como PostgreSQL-only.
- Inicializacao, pool, migracoes, backup/restore e importacao SQL adaptados para PostgreSQL.
- SQL legado de SQLite substituido por SQL compativel com PostgreSQL.
- Verificacoes de metadados migradas de PRAGMA/sqlite_master para information_schema.

## Principais Mudancas Tecnicas

### 1) Configuracao e conexao
- DATABASE_URL tornou-se obrigatoria e validada em startup.
- Removidos DB_PATH/DATABASE_PATH e logica de fallback para SQLite.
- Pool de conexoes simplificado para PostgreSQL-only com psycopg_pool.

### 2) Metadados de schema
- PRAGMA table_info, PRAGMA index_* e sqlite_master removidos.
- Introduzidos helpers centralizados em db_utils:
  - table_exists(conn, table_name)
  - get_table_columns(conn, table_name)
- Consultas de metadados agora usam information_schema.

### 3) DDL e colunas de identidade
- Trechos com INTEGER PRIMARY KEY AUTOINCREMENT migrados para:
  - INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY
- Defaults de data/hora ajustados para CURRENT_TIMESTAMP quando aplicavel.

### 4) UPSERT/REPLACE de SQLite
- INSERT OR REPLACE e REPLACE INTO substituidos por INSERT ... ON CONFLICT ... DO UPDATE.
- Mantida semantica funcional de sobrescrita por chave unica.

### 5) Backup e restore
- Fluxo de backup/restauracao padronizado para SQL PostgreSQL.
- Suporte com pg_dump/psql quando disponivel e fallback data-only interno.
- Removida migracao de arquivo .db no painel.

### 6) Importacao SQL
- Importador SQL simplificado para executar scripts no PostgreSQL configurado.
- Mantida assinatura de compatibilidade para chamadas antigas.

## Compatibilidade Funcional Preservada
- Regras de negocio em autenticacao, apostas, resultados e ranking mantidas.
- Estrategia de migracao focada em adaptacao de persistencia e metadados, sem alterar regras de dominio.

## Impactos de Configuracao
- Requisito obrigatorio: DATABASE_URL valida com esquema PostgreSQL.
- Dependencias de PostgreSQL mantidas em requirements:
  - psycopg[binary]
  - psycopg-pool

## Validacao Executada
- Busca global por marcadores de SQLite (sqlite3, sqlite_master, PRAGMA, AUTOINCREMENT, REPLACE INTO, INSERT OR REPLACE, etc.): sem ocorrencias.
- Verificacao de erros no workspace: sem erros.

## Riscos Residuais
- Ambientes sem utilitarios de cliente PostgreSQL (pg_dump/psql) usam fallback interno para backup/restore.
- Scripts SQL externos com sintaxe nao compativel com PostgreSQL podem exigir ajuste antes de importacao.
